## 需求标题
优化实时语音对话系统：实现全自动语音通话功能

## 需求背景
当前项目是一个基于 WebSocket 的实时语音对话应用，支持文本模式和语音模式。但存在以下问题：
1. **语音模式需要手动操作**：用户需要按住按钮说话，松开才发送，体验不流畅
2. **缺少自动语音检测**：无法自动检测用户何时开始/结束说话
3. **交互不自然**：无法实现像打电话一样的自然对话体验
4. **缺少实时反馈**：用户说话时没有实时的语音活动指示

**目标**：将项目优化为**类似电话通话的全自动实时语音对话系统**，用户无需手动按钮控制，系统自动检测语音活动并进行对话。

## 功能要求

### 核心功能

#### 1. **自动语音活动检测（VAD - Voice Activity Detection）**
- 系统自动检测用户何时开始说话
- 自动检测用户何时停止说话
- 去掉"按住说话"按钮，改为自动监听模式
- 支持可配置的灵敏度阈值

**技术要求**：
- 使用 OpenAI Realtime API 的 server_vad (服务端语音活动检测)
- 前端实时显示语音活动状态
- 支持调整静音检测时长（silenceDurationMs）

#### 2. **全双工语音通话**
- 用户和 AI 可以同时说话（像真实电话通话）
- 支持打断：用户可以在 AI 说话时打断
- AI 检测到用户说话时自动停止输出

**技术要求**：
- 持续音频流传输（不是分段录制）
- 实时双向音频流处理
- 低延迟（<500ms）

#### 3. **实时语音状态指示**
- **用户说话状态**：
  - 检测到语音活动时显示"正在说话"
  - 显示实时音量指示器
  - 说话结束后显示"处理中"

- **AI 说话状态**：
  - AI 开始回复时显示"AI 正在回复"
  - 实时显示 AI 音频播放进度
  - 回复结束后恢复监听状态

- **连接状态**：
  - 未连接 / 连接中 / 已连接 / 通话中 / 错误

#### 4. **一键开始/结束通话**
- **开始通话按钮**：
  - 点击后自动连接到 Realtime API
  - 自动开始音频监听
  - 进入通话模式

- **结束通话按钮**：
  - 点击后停止音频流
  - 断开 WebSocket 连接
  - 释放麦克风权限

#### 5. **音频流优化**
- 持续录音流（不间断）
- 音频缓冲优化（避免爆音、断断续续）
- 自动音量归一化
- 回声消除（如可能）

#### 6. **实时转录显示（可选）**
- 显示用户说话的实时转录文本
- 显示 AI 回复的文本内容
- 支持滚动查看历史对话

### 安全要求
- 麦克风权限请求用户授权
- WebSocket 连接使用安全传输（wss://）
- API Key 不暴露在前端代码中
- 音频数据不在本地存储

### 性能要求
- **延迟**：端到端延迟 <1s（理想 <500ms）
- **音频质量**：采样率 24kHz，16-bit PCM
- **并发**：支持单用户稳定长时间通话（>10分钟）
- **资源占用**：前端内存占用 <100MB

## UI/UX要求

### 界面设计
1. **简洁的通话界面**：
   - 大的"开始通话"按钮（未连接时）
   - 大的"结束通话"按钮（通话中）
   - 状态指示器（清晰可见）
   - 音量可视化（波形或音量条）

2. **状态反馈**：
   - 连接状态：图标 + 文字说明
   - 说话状态：实时高亮显示谁在说话
   - 错误提示：友好的错误信息

3. **可选功能区**：
   - 实时转录显示区域
   - 音量调节
   - 静音按钮

### 交互流程
```
1. 用户打开页面
2. 点击"开始通话"
3. 授权麦克风权限
4. 自动连接到 Realtime API
5. 系统提示"通话已建立，请说话"
6. 用户开始说话 → 自动检测 → 发送音频流
7. 用户停止说话 → 自动检测 → AI 开始回复
8. AI 说话 → 自动播放音频
9. 用户可以随时打断 AI
10. 点击"结束通话"断开连接
```

## 验收标准
- [ ] 用户无需按住按钮，说话自动被检测和发送
- [ ] AI 回复自动播放，无需手动操作
- [ ] 用户可以在 AI 说话时打断
- [ ] 界面实时显示语音活动状态
- [ ] 端到端延迟 <1s
- [ ] 连续通话 10 分钟无卡顿、无崩溃
- [ ] 音频播放流畅，无爆音
- [ ] 测试覆盖率 >90%
- [ ] 所有自动化测试通过

## 技术约束
- **必须使用**：OpenAI Realtime API（通过 vectorengine.ai）
- **前端**：原生 JavaScript（保持现有架构）
- **后端**：Python + aiohttp + websockets（保持现有架构）
- **音频处理**：Web Audio API（AudioWorklet）
- **浏览器兼容性**：Chrome/Edge 最新版

## 依赖项
- OpenAI Realtime API 可用性
- 用户授权麦克风权限
- WebSocket 连接稳定性

## 排除范围
- 不支持多人通话（仅 1对1 与 AI）
- 不支持视频通话
- 不支持录音保存到本地
- 不支持离线模式
